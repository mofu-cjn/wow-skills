# WCL 大秘境减伤 / 治疗复盘

帮助用户复盘一份魔兽世界 WCL（中文客户端导出）大秘境日志:在 Boss 关键技能命中点前后，
**3 个 DPS + 1 个治疗**各自开了什么减伤、自救自疗、团队减伤、外抛保护，以及治疗的预减伤与抬血是否到位。

核心目标不是堆数据,而是回答用户真正关心的两件事:
1. **各 DPS 与治疗当时开了什么减伤 / 自救?**
2. **治疗在命中前挂了什么预减伤、命中后用什么技能对谁抬血?**

然后结合伤害承受日志里的减免数据,**点出谁接得好、谁可能漏交减伤**,给出可执行的复盘要点。

---

## 固定配置(不要询问用户,直接套用)

- 大秘境 5 人队:1 坦克 / 1 治疗 / 3 DPS。
- **只复盘 3 个 DPS + 1 个治疗,不分析坦克。** 坦克减伤逻辑与队友不同,且用户明确不需要。
- 每个关键技能:复盘**命中点前后各 7 秒**的窗口(共约 14 秒)。窗口之外的技能与本次接伤无关,丢弃。

---

## 必需输入与缺失处理

每次复盘需要用户提供:

1. **施法(Cast)日志 CSV** — 列通常为:时间 / 类型 / 技能 / 来源→目标。用来看"谁开了什么技能"。
2. **伤害承受(Damage Taken)日志 CSV** — 用来**锁定 Boss 关键技能的命中事件**并读出减免数据。格式见下一节。
3. **小队专精截图** — 玩家名(职业配色)+ 职业/专精图标,用来确认每个玩家的职业专精。
4. **一个或多个关键时间点或技能名** — 可一次给多个,逐个批量复盘。

**关键约束:施法日志里没有 Boss 的伤害事件。** 关键技能的命中时间、目标、伤害量、减免效果**只能靠伤害承受日志锁定**。
所以如果用户**没给伤害承受日志**,要明确提示补充,并说明:仅凭施法日志只能按用户给的时间**近似对齐**窗口,
**无法判断减免效果是否生效**(因为看不到 U / A / M 数据),复盘结论会大幅降级为"开没开技能"的粗略层面。

如果缺专精截图,可尝试用施法日志的技能指纹反推职业专精,但要标注这是推断、可能有误。

## 伤害承受日志格式解读(务必内化)

每行大致是:

```
时间 , "Boss名 技能名 目标 [Tick] [落地数字] (A: x) (U: y, M: z, p%)"
```

字段含义——这是判断"接得好不好"的全部依据,读错会得出错误结论:

| 标记 | 含义 | 复盘里怎么用 |
|---|---|---|
| **落地数字**(裸整数) | 实际掉血量 | 真正打进血量的伤害。数字越大、减免越低 = 接得越差 |
| 只有 **(A: x)** 没有落地数字 | 这一下被**完全吸收/减免**,没真正掉血 | 盾或减伤把整下吃掉了,接得好 |
| **Tick** | 这一行是 **DoT 持续伤害的一跳**(非直击) | 判断是直击还是持续伤害;DoT 适合用持续型减伤/团减覆盖 |
| 没有 **Tick** | **直击 / 瞬时伤害** | 大额直击更依赖在命中点交个人减伤 |
| **A (absorbed)** | 被吸收的量(护盾 / 外抛盾) | A 占比大 = 有盾接住了(可能自己的盾,也可能外抛) |
| **U (unmitigated)** | 未经减免的**原始伤害**(本来多大) | 和落地数字对比,看实际削减了多少 |
| **M (mitigated, p%)** | 被减免的量 + **减免百分比 p%** | **最关键的单一指标。** p% 高 = 减伤/盾接上了;p% 很低又裸吃大额直击 = 多半没交个人减伤 |
| **Dodge / Parry / Miss / Immune** | 该下被闪避 / 招架 / 未命中 / 免疫 | 未造成伤害,免疫往往是主动技能(如假死、圣盾、暗影斗篷),要算作"接得好"并归因 |

判断逻辑要把 **M%** 和 **A:** 当成核心证据:不要光看玩家开没开技能,要用减免数据**反推**技能到底有没有生效、生效程度如何。

## 分析流程(对每个关键技能依次执行)

### 第 0 步:专精识别(整份只做一次)

用截图(**名字颜色 = 职业,图标 = 专精**)+ 施法日志技能指纹**交叉验证**,输出"玩家 | 职业·专精 | 角色"确认表。
交叉验证很重要:截图可能模糊或图标相近,而某些技能是专精独有的指纹(例如只有戒律牧才会读"真言术:盾"+"苦修"组合),
两边对上才敢下结论。识别一次后,后续所有时间点复用同一张表。

### 第 1 步:用伤害承受日志锁定关键技能

在用户给的时间点附近,定位该 Boss 技能的伤害事件,提取:**精确时间、技能名、伤害类型、命中目标、直击还是 DoT、
落地量、U / A / M%**。

伤害类型由**技能名推断**(物理 / 魔法 / 火焰 / 冰霜 / 自然 / 暗影 / 奥术等)。类型决定后面给什么建议——
例如魔法尖刺类伤害,该用法系减伤或魔法团减提前铺,物理减伤帮助有限。

### 第 2 步:读减免,判断每个被命中角色"当时实际接得好不好"

对每个被命中的复盘对象:
- **减免% 高 / 吸收(A)占比大** → 减伤或盾接上了。
- **减免% 很低且裸吃大额直击** → 多半没交个人减伤。**回到施法日志核实**该角色窗口内有没有相应技能。
- 被 **Immune / Dodge** 等 → 归因到对应主动技能(免疫类常常就是关键自保)。

### 第 3 步:过滤施法日志窗口(命中点 ±7s)

只保留与接伤相关的技能:**减伤 / 自救自疗 / 团队减伤 / 外抛保护 / 治疗 / 位移自保**。
**丢弃纯输出循环与噪音**:攻击技能、资源生成(灵魂残片、连击点、符文能量等)、增益维持类输出 buff。
分类时以 `references/ability-categories.md` 为准(见下文),表里没有的再凭功能推断并标注。

### 第 4 步:排时间线

按时间顺序列出窗口内保留的技能,**中间用单独一行标注关键技能命中点**,把事件清晰分成"命中前 / 命中后"。

### 第 5 步:回答两问

① 各 DPS 与治疗开了什么减伤 / 自救?
② 治疗:命中**前**挂了什么预减伤(护盾、预读条、团减)?命中**后**用什么技能、**对谁**抬血?

### 第 6 步:简评 + 复盘要点

结合第 2 步的减免数据给评价:谁接得稳、谁可能漏交、治疗节奏是否合理。建议要具体、可执行,
并结合伤害类型(如"这是魔法直击,但 XX 没提前铺法系减伤")。

---

## 多时间点批量处理

用户一次给多个时间点 / 技能时:

1. **逐个输出独立小节**:每节包含 锁定信息 + 时间线 + 两问作答 + 简评(完整走一遍上面的流程)。
2. 全部做完后,**补一段"跨时间点总结"**:指出反复出现的问题,例如——
   - 同一角色每次都裸吃同类伤害;
   - 治疗的预减伤是否每次都到位;
   - 某个团队减伤 / 外抛是否每次都漏开;
   - 哪些是个人操作问题,哪些可能是机制/读取问题。
   然后给整体优化建议(谁该练什么、团队减伤怎么分配)。

跨时间点总结是批量复盘最有价值的部分——单点看不出的习惯性问题,放在一起才暴露出来。

## 判定规则(避免误判)

这些区分直接决定结论对不对,逐条对照:

- **减伤/自疗 ≠ 位移/读条保护。** 唤魔师悬空、武僧滚地翻 = 位移自保,**不算减伤**(它们不削减承受的伤害,只是规避/走位)。
- **区分四类生存技能**:个人减伤(只保自己) / 团队减伤(保全队) / 外抛保护(给别人的盾或减伤) / 治疗(回血)。
  外抛要看技能的**目标**——同一个盾,给自己是个人,给队友是外抛。
- **辅助 buff ≠ 减伤。** 增辉的先知先觉、黑檀之力等是**增伤/辅助**,不是减伤,不要算进生存里。
- **保留中文技能名**,可在括号里补功能说明;**不确定的英文名不要硬翻**。
- 被命中角色在窗口内**若无个人减伤、且减免% 很低**,明确点名,并提示**两种可能**:
  (a) 操作失误漏交;(b) 技能未导出到 CSV(WCL 偶尔漏记某些 buff)。两种都要提,不要武断只说失误。
- **结合伤害类型给建议**(物理 / 魔法分别该用什么减伤),不要泛泛说"多交减伤"。

## 技能分类参考文件

`references/ability-categories.md` 是一张**中文技能名 → 类别**(个人减伤 / 自疗 / 团减 / 外抛 / 位移自保 / 辅助 / 输出)
的分类表,按职业专精组织,覆盖当前版本(地心之战 / 午夜,11.x–12.x)常见 DPS 与治疗专精。

**用法**:在第 3 步过滤施法日志和第 5 步归类时,**先查表**确定类别,降低误判。
表里没收录的技能,再凭功能推断并**标注"(推断)"**。

这张表是**起点而非铁律**:版本更新会增删技能、改名,英雄天赋还会带来新主动技能。
遇到与表冲突的实际情况,以日志+功能判断为准,并可顺手向用户指出表该更新的地方。

## 输出格式

全程**中文输出**。结构如下:

**1. 专精确认表**

| 玩家 | 职业·专精 | 角色 | 判定依据 |
|---|---|---|---|

**2. 每个关键技能一节**

- **a. 锁定信息**:时间 / 技能 / 伤害类型 / 命中目标 / 直击或 DoT / 落地量 / U·M%·A。
- **b. 时间线表**(按时间排序,中间一行标命中点):

  | 时间 | 玩家(专精) | 技能 | 目标 | 性质 / 点评 |
  |---|---|---|---|---|

  其中"性质"用分类表的类别(个人减伤 / 团减 / 外抛 / 自疗 / 治疗 / 位移 / 辅助)。
- **c. 两问作答 + 简评**。

**3. 跨时间点总结**(仅当有多个时间点时)。

输出以"回答用户的两问 + 可执行复盘要点"为重心,避免堆砌无关数据。
